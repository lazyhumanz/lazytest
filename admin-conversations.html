Participated Conversations

Count of unique conversations where the admin created at least one conversation part (message/reply) on the selected date.

How it's calculated:
  1. Fetch conversations from Intercom where admin is assigned and conversation was created or updated on the selected date
  2. For each conversation, check all conversation parts
  3. Count conversations where at least one part has:
    • Author type = "admin"
    • Author ID = admin's ID
    • Created timestamp falls within the selected date
Example:

If admin "John" created 3 messages in Conversation A and 2 messages in Conversation B on Nov 4, 2025, the count is 2 conversations (not 5 messages).

Total Parts Created

Total number of conversation parts (messages/replies) created by the admin on the selected date across all conversations.

How it's calculated:
  1. For each participated conversation, examine all conversation parts
  2. Count parts where:
    • Author type = "admin"
    • Author ID = admin's ID
    • Created timestamp falls within the selected date (00:00:00 to 23:59:59)
  3. Sum all parts across all conversations
Example:

If admin "John" created 3 messages in Conversation A and 2 messages in Conversation B on Nov 4, 2025, the total is 5 parts.

Processed

Number of conversations that were successfully fetched and analyzed from Intercom API to check for admin participation.

How it's calculated:
  1. Intercom Search API returns conversation IDs matching the date filter
  2. Each conversation is fetched individually to get its conversation parts
  3. Each conversation is analyzed to check if admin participated
  4. Count of successfully processed conversations (excluding errors)
Example:

If Intercom returned 29 conversation IDs, and all 29 were successfully fetched and analyzed, the processed count is 29.

Total from Intercom

Total number of conversations returned by Intercom Search API that match the date filter criteria (created or updated on selected date).

How it's calculated:
  1. Query Intercom Search API with filters:
    • teammate_ids = admin's ID
    • created_at OR updated_at within selected date range
  2. Intercom returns conversation IDs (up to 150 per page)
  3. This count represents the raw total from Intercom before participation filtering
Example:

If Intercom Search API returns 29 conversation IDs for the selected date, this count is 29. Note: Not all of these may have admin participation.
